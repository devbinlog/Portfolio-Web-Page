# 01. 요구사항 정의

## 기능 요구사항

### 공개 (Public)

랜딩 페이지
- 3D 월드 히어로 섹션 표시
- Identity Core 오브젝트 표시
- Featured Project 오브젝트 표시
- Taebin Kim 이름, 역할, 철학 라인 표시
- 내비게이션 (어바웃, 프로젝트, 연락처)
- 다크/라이트 테마 토글
- 인터랙션 힌트 표시

About 섹션/페이지
- 개인 소개 (이름, 역할, 배경)
- 배경 요약
- 강점 목록
- 일하는 방식 블록 (필수)
  - 문제를 기능 단위가 아닌 구조적으로 분해
  - 구현 전 문서와 아키텍처 정의
  - 데이터 구조와 사용자 경험을 함께 설계
  - 배포, 유지보수, 운영을 처음부터 고려

스킬 섹션
- 기술 스택 표시 (카테고리 구분)
- 프론트엔드, 백엔드, 인프라, 3D 기술 포함

대표 프로젝트
- Featured Project 오브젝트 3D 표시
- 프로젝트 hover 시 레이블 표시
- 프로젝트 click 시 카메라 이동 및 overlay 열기

프로젝트 overlay
- 프로젝트 제목, 요약, 설명
- 카테고리, 연도, 태그
- 역할 및 기여 내역
- 기술 스택 목록
- 갤러리 이미지
- 영상 플레이스홀더 (업로드 전 상태 명확히 표시)
- 문서/보고서 플레이스홀더 (업로드 전 상태 명확히 표시)
- GitHub 링크, 데모 링크, 기타 외부 링크
- 주요 학습 내용
- 상세 페이지 이동 링크

프로젝트 상세 페이지 (라우트 기반)
- /projects/[slug] 경로
- overlay 콘텐츠 전체 포함
- SEO 최적화된 메타데이터
- 전체 읽기 모드

프로젝트 아카이브
- 전체 프로젝트 목록
- 카테고리 필터
- 카드 형식 또는 리스트 형식

경험/활동 섹션
- 경력, 교육, 활동 이력

연락처 폼
- 이름, 이메일, 메시지 입력
- rate limit 적용 (Redis)
- 제출 성공/실패 피드백

내비게이션
- 반응형 내비게이션
- 모바일 메뉴

SEO
- 각 페이지 메타데이터
- Open Graph 태그
- sitemap.xml
- robots.txt

### 관리자 (Admin)

관리자 로그인
- JWT 기반 인증
- 로그인 시도 제한 (Redis)

대시보드
- 프로젝트 수, 연락 메시지 수 요약

프로젝트 관리
- 프로젝트 생성, 수정, 삭제
- 프로젝트 공개/비공개 전환
- Featured 순서 조정
- 슬러그 자동 생성 및 수정

미디어 관리
- 이미지 업로드 및 순서 관리
- 영상 플레이스홀더 메타데이터 관리
- 문서/보고서 플레이스홀더 메타데이터 관리
- 실제 파일 업로드로 플레이스홀더 교체

링크 관리
- GitHub, 데모, 문서, 외부 링크 CRUD

연락 메시지 목록
- 수신된 메시지 목록 조회
- 읽음/안읽음 상태 관리

## 비기능 요구사항

성능
- Lighthouse Performance 90+
- LCP 2.5초 이하
- 3D 씬 lazy loading
- 모바일에서 2D fallback으로 빠른 초기 로딩

접근성

- 키보드 내비게이션 지원
- 스크린리더 대응 (aria 속성)
- 충분한 색상 대비 (WCAG AA)

SEO
- Lighthouse SEO 90+
- 모든 페이지 고유 메타데이터
- 구조화된 데이터 (선택적)

보안
- JWT 토큰 기반 관리자 인증
- 관리자 엔드포인트 접근 제어
- 파일 업로드 검증
- rate limiting
- 환경변수로만 시크릿 관리

유지보수성
- 컴포넌트 단위 책임 분리
- 문서 우선 개발 방식
- 코드와 문서의 일관성 유지

배포 가능성
- Docker 기반 컨테이너화
- 환경 분리 (개발/스테이징/프로덕션)
- CI/CD 파이프라인
- provider-agnostic 배포 전략

## 사용자 플로우

### 일반 사용자 플로우

1. 방문자가 사이트에 접속한다
2. intro 애니메이션 후 3D 월드가 표시된다
3. Taebin Kim의 이름과 역할이 즉시 확인된다
4. 방문자가 3D 오브젝트에 마우스를 올리면 프로젝트 레이블이 표시된다
5. 오브젝트를 클릭하면 카메라가 해당 프로젝트로 이동하고 overlay가 열린다
6. overlay에서 프로젝트 정보를 탐색한다
7. 영상/문서 플레이스홀더가 있으면 준비 중임이 명확히 표시된다
8. "자세히 보기" 링크로 상세 페이지(/projects/[slug])로 이동할 수 있다
9. 뒤로 가기 또는 닫기 버튼으로 세계로 돌아간다

### 모바일 사용자 플로우

1. 방문자가 사이트에 접속한다
2. 3D 없이 2D 레이아웃의 랜딩 화면이 표시된다
3. Taebin Kim의 이름, 역할, 철학 라인이 표시된다
4. 프로젝트 카드 목록 또는 슬라이더가 표시된다
5. 카드를 탭하면 상세 overlay가 열린다
6. 나머지 플로우는 데스크톱과 동일

### 관리자 플로우

1. 관리자가 /admin/login으로 접속한다
2. 이메일/패스워드로 로그인한다
3. 대시보드에서 전체 현황을 확인한다
4. 프로젝트를 생성/수정/삭제한다
5. 미디어를 업로드하거나 플레이스홀더 메타데이터를 입력한다
6. 연락 메시지를 확인한다
7. 로그아웃한다

## 디바이스 전략

데스크톱 (1024px 이상)
- 전체 3D 월드 경험
- 마우스 기반 인터랙션
- 전체 overlay 시스템

태블릿 (768px ~ 1023px)
- 경량화된 3D + 강화된 overlay 안내
- 터치 기반 인터랙션
- 단순화된 카메라 동작

모바일 (767px 이하)
- 2D fallback 우선
- 3D 씬 비로딩
- 동일한 콘텐츠 모델, 카드/슬라이더 기반 표시
- 빠른 초기 로딩 보장

## 접근성 및 Fallback 요구사항

감소된 모션 모드 (prefers-reduced-motion)
- 3D 애니메이션 최소화
- Framer Motion 트랜지션 단순화
- 플로팅 오브젝트 모션 제거

성능 모드
- 수동 토글 지원
- 3D 렌더링 품질 하향
- 파티클/이펙트 비활성화

3D 렌더링 실패 fallback
- WebGL 미지원 환경 감지
- 2D 레이아웃으로 자동 전환
- 동일한 콘텐츠 접근성 보장

키보드 내비게이션
- Tab 키로 모든 인터랙티브 요소 접근
- 3D 오브젝트와 동일한 기능을 키보드로 수행 가능
- Skip to main content 링크 제공

## 다크모드/라이트모드 요구사항

기본값
- 시스템 기본 설정(prefers-color-scheme) 감지
- 감지 실패 시 다크 모드를 기본값으로 사용

전환
- 즉시 전환 (flash 없음)
- localStorage에 사용자 선택 저장
- 새로고침 후에도 선택이 유지됨

디자인 요구사항
- 두 테마 모두 동일한 계층 구조와 가독성을 유지
- 모든 색상은 시맨틱 토큰을 통해 전환됨
- 3D 씬의 조명과 material이 테마에 따라 조정됨
- overlay, 패널, border, text, focus 상태 모두 적응

관리 방식
- Tailwind CSS의 dark 클래스 또는 CSS 변수 기반
- next-themes 라이브러리 활용 검토
